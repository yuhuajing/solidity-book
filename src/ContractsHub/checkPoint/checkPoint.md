## checkPoint 检查点

通过创建结构体数组，用结构体数组的方式记录当前链的状态，用于查询/更新特定键值或新增键值，表示对于当前状态的操作

1. 状态快照：通过记录关键状态的快照，允许合约在特定时间点恢复到先前的状态。这在需要支持撤销、回滚或跟踪历史状态的应用中很有用。

2. 安全性保证：通过引入检查点机制，确保在复杂操作期间发生异常或失败时，可以恢复到操作之前的状态。这可以提供更强的安全性和可靠性，避免中间状态对系统造成损害。

3. 执行步骤跟踪：通过在关键步骤处创建检查点，可以跟踪合约的执行路径，以便在调试或分析代码时更好地理解和排查问题。

其中，lowerLookup返回第一个大于等于 key 的 value值

upperLookup返回第一个小于等于 key 的value值

        return _insert(self._checkpoints, key, value);
    }

     * @dev Returns the value in the first (oldest) checkpoint with key greater or equal than the search key, or zero if there is none.

        uint256 len = self._checkpoints.length;
        uint256 pos = _lowerBinaryLookup(self._checkpoints, key, 0, len);
        return pos == len ? 0 : _unsafeAccess(self._checkpoints, pos)._value;
